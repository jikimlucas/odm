Welcome to OpenDroneMap.

Projects like Google Streetview, Mapillary, PhotoSynth, and most small UAS (drone) postprocessing software, such as that offered by senseFly, share a commonality-- they all use computer vision techniques to create spatial data from un-referenced photography.

OpenDroneMap is an open source project to unlock and make easy-to-use computer vision techniques, so that whether from street level photos or from civilian drone imagery, the average user will be able to generate point clouds, 3D surfaces models, DEMs, and orthophotography from processing unreferenced photos.

The first step in ODM is to build the toolchain for processing data. Long term, the aim is to create an open repository for civilian drone and surface level data, in the spirit of OpenStreetMapâ€‹.

For the moment, OpenDroneMap will use an existing FOSS toolchain with Bundler SfM, Patch-based Multi-view Stereo Software, Clustering Views for Multi-view Stereo, and related tools. The aim will be to wrap this toolchain in a simple to use and easy to deploy web interface in order to bootstrap the project.

In the near future, the project may use more liberally licensed software, such as Libmv.

Dependencies:

Currently the project is deployed using Vagrant using a 64bit Ubuntu VM. To get running, install Vagrant and VirtualBox. Clone or download this repository. Navigate to the directory and run:

vagrant up

It will take some time for the machine to download, compile, and install all the software. Directions on how to use are forthcoming.
